## 四条核心理念

| 理念 | 大白话解释 |
|:----|:----------|
| **测试驱动开发** | 永远先写测试，再写代码，代码通过测试才算完成 |
| **系统性胜过随意** | 按流程办事，不靠蒙，不靠"感觉应该没问题" |
| **化简为繁** | 简单是首要目标，能用简单方案就不用复杂方案 |
| **证据胜过口头保证** | 验证通过才算完成，不说"应该好了"要"已经验证过了" |

## 7阶段工作流程

Stage 1: Brainstorming（设计对话）
Stage 2: Using Git Worktrees（隔离分支）
Stage 3: Writing Plans（计划策定）
Stage 4: Sub-Agent Driven Development（子代理驱动）
Stage 5: Test-Driven Development（测试驱动开发）
Stage 6: Code Review（代码审查）
Stage 7: Finishing a Development Branch（完成处理）

## 15个子技能

### 协作类
- brainstorming -- 通过提问、方案评估和增量验证进行协作设计
- writing-plans -- 小粒度实施计划（精确文件路径 + TDD 步骤）
- executing-plans -- 带检查点的计划批量执行
- subagent-driven-development -- 每个任务新鲜子代理 + 两阶段审查
- dispatching-parallel-agents -- 分发独立任务给并发代理

### 测试/调试类
- test-driven-development -- RED-GREEN-REFACTOR 循环
- systematic-debugging -- 4阶段根本原因分析
- verification-before-completion -- 运行命令->读取输出->声称结果

### 代码审查类
- requesting-code-review -- 分发并分类反馈
- receiving-code-review -- 验证反馈，不盲目同意

### Git 相关
- using-git-worktrees -- 隔离 git worktree
- finishing-a-development-branch -- 合并/PR/清理

### 元技能
- writing-skills -- 创建/测试/部署新技能
- using-superpowers -- 技能系统使用指南
