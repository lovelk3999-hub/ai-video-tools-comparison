## 一、项目定位

**做什么：** AI 视频制作工具的费用对比站（程序化 SEO）
**目标用户：** 想用 AI 做视频但不知道选哪个工具的人

这是我们整个程序化 SEO 方向的 **原始灵感来源**。

| 维度 | 数据 |
|:----|:-----|
| 网站类型 | 烘焙原料替代计算器/对比站 |

**对我们的启示：**
- 程序化 SEO 需要耐心（前 5-6 个月零流量是正常的）

- 选择有对比/替换需求的品类（烘焙原料替代 -> AI视频工具对比）
- 成本极低（$10/年域名 + 免费托管），放着等积累就行

## 二、竞品调研

### 2.1 LLM API 定价对比站（参考对象）

在决定做 AI 视频工具之前，先调研了 LLM API 定价对比赛道的竞品，了解这类网站能做到什么程度。

| 网站 | 月流量 | 页面数 | 深度评级 | 策略 |
|:----|:-----:|:------:|:--------:|:-----|
| **Artificial Analysis** | 50万-100万 | 50+ | ★★★★★ | 多维度排行榜 + 交互筛选器 + 多赛道（文本/图片/语音） |
| **CostGoat** | 13.9万 ⬆️ | 20+ | ★★★★ | 每个服务一个独立定价计算器，程序化SEO思路 |
| **PricePerToken** | 4.5-6万 | 300+ | ★★★★ | 每个模型一个独立页面，吃长尾流量 |
| **MorphLLM** | 未知 | 15+ | ★★★ | 定价表 + 计算器 + 优化指南 |

**关键发现：**
- 这个赛道已经有成功案例，但**AI视频工具对比站还没有入局者**
- CostGoat 的模式和我们最接近：程序化生成 + 独立计算器页面
- PricePerToken 的"每个模型一个独立页"策略说明长尾SEO有效
- Artificial Analysis 是终极目标，但起步不需要做那么重

### 2.2 已发现的 AI 视频工具对比内容

| 类型 | 例子 | 特点 |
|:----|:-----|:-----|
| 大媒体综述 | CNET / Zapier / MASV | 每年一篇，内容宽泛，无定价对比 |
| 工具商自比 | HeyGen 官网对比页 / Colossyan 博客 | 有偏向性 |
| 单篇博文 | comparebestai.com / digitallinkage.com | 手工写，不批量 |
| 社区内容 | Reddit / Medium / YouTube | 零散，不系统 |

**结论：没有人做 AI 视频工具的**结构化对比站 **，蓝海。**

## 三、信息采集策略

### 3.1 各工具官网反爬情况实测

| 工具 | 普通请求 | Playwright | 反爬难度 |
|:----|:--------:|:----------:|:--------:|
| **Runway** | ✅ 200 SSG | — | 🟢 简单，requests 即可 |
| **Pika** | ✅ 200 SSR | — | 🟢 简单，requests 即可 |
| **HeyGen** | ✅ 200, 773KB | — | 🟢 简单，requests 即可 |
| **Kling** | ❌ 405 | ✅ 绕过 | 🟡 中等，Playwright 能过 |
| **Synthesia** | ❌ 403 WAF | ✅ 绕过 | 🟡 中等，Playwright 能过 |

**结论：** 这些是商业SaaS官网，反爬很弱，Playwright 无头浏览器通行无阻。不需要代理池/Cookie管理/专业爬虫。

### 3.2 推荐采集架构

```
起步（Level 2）：
  手动查官网 → 填 tools.json → 跑 generate.py → 部署

未来（Level 3，网站有流量后再加）：
  GitHub Actions (每月1次)
    → Playwright 采集5个定价页
    → 对比上次数据
    → 有变化 → 重新生成 → 自动部署
    → 没变化 → 跳过
```

### 3.3 更新频率建议

| 内容类型 | 更新频率 | 方式 |
|:--------|:--------|:-----|
| 定价数据 | 每月1次 | 手动或自动检查 |
| 新工具 | 按需 | 发现新工具就加 |
| 功能对比维度 | 每季度 | 关注各工具Changelog |
| 用户反馈/评测 | 每月 | agent-reach 搜 Reddit/小红书等 |

## 四、搭建方案（三档对比）

### Level 1：纯静态（❌ 不推荐）
```
手写6-10个HTML，数据写死
扩展性差，加个工具要改所有页面
```

### ✅ Level 2：模板 + 生成器（推荐起步）
```
data/tools.json     ← 核心数据，唯一需要手动维护
templates/*.html    ← 3个模板（首页/工具页/对比页）
scripts/generate.py ← 一个命令生成所有页面

上线时间：1-2天
月维护时间：≈15分钟
扩展能力：可到500+页面
```

### Level 3：全自动采集（暂缓）
```
Level 2 + GitHub Actions + Playwright 自动爬价格
月维护时间不确定（爬虫可能因网站改版而挂）
等网站有流量验证后再加
```
