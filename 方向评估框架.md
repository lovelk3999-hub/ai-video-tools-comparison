# 程序化 SEO 方向评估框架

> 创建日期：2026-06-19
> 用途：系统性地评估某个方向是否值得用程序化SEO去做

---

## 8 维评估体系

### 1. 搜索量（TAM）
| 评分 | 含义 |
|:----:|:-----|
| 5 | 月搜索量 100万+（大品类） |
| 4 | 月搜索量 10万-100万（中品类） |
| 3 | 月搜索量 1万-10万（小品类） |
| 2 | 月搜索量 1千-1万（细分） |
| 1 | 月搜索量 <1千（冷门） |

### 2. 数据可结构化程度
| 评分 | 含义 |
|:----:|:-----|
| 5 | 多维属性丰富（10+维度），可批量生成 200+ 页面 |
| 4 | 多维属性较丰富（6-9维度），可生成 100-200 页面 |
| 3 | 有明确维度（3-5维度），可生成 50-100 页面 |
| 2 | 维度少（1-2维度），页面区分度低 |
| 1 | 基本无法结构化，只能手写 |

### 3. 竞争格局
| 评分 | 含义 |
|:----:|:-----|
| 5 | 无人用程序化SEO做，大媒体不覆盖 |
| 4 | 少量竞品但不成熟 |
| 3 | 有2-3个同类站 |
| 2 | 多个成熟竞品 |
| 1 | 红海，头部已形成 |

### 4. 商业价值
| 评分 | 含义 |
|:----:|:-----|
| 5 | 高客单价 + 高佣金（$50+/单） |
| 4 | 中客单价 + 有联盟计划（$10-50/单） |
| 3 | 低客单价 + AdSense为主 |
| 2 | 盈利模式不确定 |
| 1 | 几乎无法变现 |

### 5. 数据采集难度
| 评分 | 含义 |
|:----:|:-----|
| 5 | 公开API或简单requests即可 |
| 4 | Playwright无头可过 |
| 3 | 需要登录/反爬绕行（TinyFish级别） |
| 2 | 需要大量人工处理 |
| 1 | 几乎无法稳定采集 |

### 6. 信息更新频率
| 评分 | 含义 |
|:----:|:-----|
| 5 | 一年以上才变一次（一次投入长期收益） |
| 4 | 每季度变一次 |
| 3 | 每月变一次 |
| 2 | 每周变一次（维护成本高） |
| 1 | 实时变动 |

### 7. 你的了解程度
| 评分 | 含义 |
|:----:|:-----|
| 5 | 你就是目标用户，知道数据在哪 |
| 4 | 有所了解，查一下就能上手 |
| 3 | 听说过但不熟 |
| 2 | 需要花时间学习 |
| 1 | 完全陌生 |

### 8. Google友好度
| 评分 | 含义 |
|:----:|:-----|
| 5 | 对比/比价类 -> Google明确欢迎 |
| 4 | 信息聚合类 -> 有独特数据来源 |
| 3 | 推荐/评测类 -> 需要原创内容 |
| 2 | 纯affiliate类 -> 容易被判定低质量 |
| 1 | 搜索没意义 -> 用户不会这样搜 |

---

## 评分公式

```
总得分 = (搜索量x1.5) + (结构化x1.5) + (竞争x1.5) + (商业x1.2) + (采集x1.0) + (更新x0.8) + (熟悉x0.8) + (Googlex1.5)
```

**权重说明：**
- 搜索量/结构化/竞争/Google友好度：最重要的维度
- 商业价值：重要，最终要赚钱
- 采集/更新/熟悉度：辅助维度，影响启动和运维成本

**总分解读：**
| 分数区间 | 结论 |
|:--------:|:-----|
| 55+ | 强烈推荐，立即启动 |
| 45-54 | 值得做，可以启动 |
| 35-44 | 可以考虑，需要仔细评估 |
| 25-34 | 不推荐 |
| <25 | 跳过 |
